  <action>
Create `static/svg-shapes.js` as an IIFE exposing `window.SmartBSvgShapes`:

1. **SVG namespace constant**: `var NS = 'http://www.w3.org/2000/svg';`

2. **Shape rendering functions** -- each takes `(w, h)` (width, height) and returns an SVG element centered at origin (0,0). All shapes draw from (-w/2, -h/2) to (w/2, h/2) so they work with dagre's center-point positioning via `translate(x,y)`:

   - `rect(w, h)`: `<rect>` at (-w/2, -h/2), rx=0
   - `rounded(w, h)`: `<rect>` at (-w/2, -h/2), rx=8
   - `stadium(w, h)`: `<rect>` at (-w/2, -h/2), rx=h/2 (fully rounded ends)
   - `diamond(w, h)`: `<polygon>` with points at top(0,-h/2), right(w/2,0), bottom(0,h/2), left(-w/2,0)
   - `circle(w, h)`: `<circle>` with r=max(w,h)/2
   - `hexagon(w, h)`: `<polygon>` 6-point hexagon. Indent dx = h/4. Points: top-left(-w/2+dx, -h/2), top-right(w/2-dx, -h/2), right(w/2, 0), bottom-right(w/2-dx, h/2), bottom-left(-w/2+dx, h/2), left(-w/2, 0)
   - `subroutine(w, h)`: `<g>` containing: main `<rect>` at (-w/2, -h/2) with rx=0, plus two vertical lines at x=-w/2+8 and x=w/2-8 (the double-border effect)
   - `cylinder(w, h)`: `<path>` drawing a cylinder shape. Top ellipse arc, sides, bottom ellipse arc. The cylinder has an extra 8px on top for the cap visual.
   - `asymmetric(w, h)`: `<polygon>` flag/pennant shape. Points: top-left(-w/2, -h/2), top-right(w/2, -h/2), point(w/2 - h/4, 0), bottom-right(w/2, h/2), bottom-left(-w/2, h/2)
   - `parallelogram(w, h)`: `<polygon>` skewed right. Skew dx = w * 0.15. Points: top-left(-w/2+dx, -h/2), top-right(w/2+dx, -h/2), bottom-right(w/2-dx, h/2), bottom-left(-w/2-dx, h/2). Wait -- Mermaid parallelogram `/text/` slants right. The correct points: top-left(-w/2 + skew, -h/2), top-right(w/2, -h/2), bottom-right(w/2 - skew, h/2), bottom-left(-w/2, h/2). Use skew = h * 0.3.
   - `parallelogram-alt(w, h)`: `<polygon>` slanting left (mirror of parallelogram). Points: top-left(-w/2, -h/2), top-right(w/2 - skew, -h/2), bottom-right(w/2, h/2), bottom-left(-w/2 + skew, h/2). Same skew = h * 0.3.
   - `trapezoid(w, h)`: `<polygon>` wider at bottom. Indent dx = w * 0.15. Points: top-left(-w/2+dx, -h/2), top-right(w/2-dx, -h/2), bottom-right(w/2, h/2), bottom-left(-w/2, h/2)
   - `trapezoid-alt(w, h)`: `<polygon>` wider at top (inverted trapezoid). Points: top-left(-w/2, -h/2), top-right(w/2, -h/2), bottom-right(w/2-dx, h/2), bottom-left(-w/2+dx, h/2)

3. **Shape registry**: `var SHAPE_RENDERERS = { rect: rect, rounded: rounded, ... }` mapping all 13 shape names.

4. **render(shape, w, h) function**: looks up `SHAPE_RENDERERS[shape]`, falls back to `rect` if unknown, calls the function with `(w, h)`, returns the SVG element.

5. **Public API**: `window.SmartBSvgShapes = { render: render }`

Use `var` throughout. Use `'use strict'` in IIFE. Every SVG element must use `document.createElementNS(NS, tagName)` -- never `document.createElement()`. The file should be ~200-280 lines.

**Style note:** Do NOT apply fill/stroke to shapes -- the renderer (svg-renderer.js) handles styling after receiving the shape element. Shapes are geometry-only.

<task type="auto">
  <name>Task 2: Create svg-renderer.js for complete SVG DOM assembly</name>
  <files>static/svg-renderer.js</files>
  <action>
Create `static/svg-renderer.js` as an IIFE exposing `window.SmartBSvgRenderer`:

1. **SVG namespace**: `var NS = 'http://www.w3.org/2000/svg';`

2. **Theme constants** (matching renderer.js MERMAID_CONFIG themeVariables):
   ```
   var THEME = {
       nodeFill: '#eef2ff',
       nodeStroke: '#4f46e5',
       nodeStrokeWidth: '2',
       nodeTextColor: '#1e1b4b',
       nodeFontSize: '15',
       nodeFontWeight: '600',
       nodeFontFamily: "'Inter', sans-serif",
       edgeStroke: '#6b7280',
       edgeStrokeWidth: '1.5',
       edgeLabelBg: '#ffffff',
       edgeLabelColor: '#6b7280',
       edgeLabelFontSize: '13',
       subgraphFill: '#f8fafc',
       subgraphStroke: '#cbd5e1',
       subgraphStrokeWidth: '1',
       subgraphLabelColor: '#1e293b',
       subgraphLabelFontSize: '14',
       subgraphLabelFontWeight: '700',
   };
   ```

3. **createArrowMarkers(defs)**: Creates SVG `<marker>` elements in a `<defs>` block:
   - `arrow-normal`: viewBox "0 0 10 10", refX=10, refY=5, size=8, polygon "0 0, 10 5, 0 10", fill=#6b7280
   - `arrow-thick`: same but size=10
   - `arrow-dotted`: same as normal (dashed styling is on the path, not the marker)

4. **edgePointsToPath(points)**: Converts dagre edge control points array into SVG path `d` attribute.
   - Start with `M points[0].x points[0].y`
   - If exactly 2 points: simple `L` line
   - If 3+ points: use cubic bezier `C` commands. Group intermediate points into triplets for C command control points. If remaining points don't form complete triplets, finish with `L` to last point.

5. **shortenPathToNodeBoundary(points, targetNode)**: Shorten the last edge segment so arrowhead doesn't hide inside the target node.
   - For `rect`, `rounded`, `stadium`, `subroutine`: clamp to rectangle boundary (x +/- w/2, y +/- h/2)
   - For `circle`: offset by radius along the direction vector
   - For `diamond`: line-diamond intersection
   - For other shapes: approximate with rectangle boundary (good enough for visual quality)
   - Adjust the last point in the `points` array, return modified array

6. **renderNode(node)**: Creates a `<g>` group for a single node:
   - `g.setAttribute('data-node-id', node.id)`
   - `g.setAttribute('class', 'smartb-node')`
   - `g.setAttribute('transform', 'translate(' + node.x + ',' + node.y + ')')`
   - Call `SmartBSvgShapes.render(node.shape, node.width, node.height)` for the shape element
   - Apply THEME styles to shape: fill, stroke, stroke-width
   - Create `<text>` element: text-anchor=middle, dominant-baseline=central, fill, font-family, font-size, font-weight from THEME. textContent = node.label
   - Append shape and text to g, return g

7. **renderEdge(edge)**: Creates a `<g>` group for a single edge:
   - `g.setAttribute('data-edge-id', edge.id)`
   - `g.setAttribute('class', 'smartb-edge')`
   - Shorten path to target boundary using `shortenPathToNodeBoundary`
   - Create `<path>` with `d` from `edgePointsToPath`, fill=none
   - Apply edge type styling:
     - `arrow`: stroke-width=1.5, marker-end=url(#arrow-normal)
     - `thick`: stroke-width=3, marker-end=url(#arrow-thick)
     - `dotted`: stroke-width=1.5, stroke-dasharray=5,5, marker-end=url(#arrow-normal)
     - `open`: stroke-width=1.5, no marker-end
     - `invisible`: stroke=none
   - If edge has a label: create a `<g>` for label positioned at the midpoint of the edge. Background `<rect>` (white, small padding) + `<text>`.
   - Append path (and optional label group) to g, return g

8. **renderSubgraph(sg)**: Creates a `<g>` group for a subgraph:
   - `g.setAttribute('data-subgraph-id', sg.id)`
   - `g.setAttribute('class', 'smartb-subgraph')`
   - Background `<rect>` at (x - w/2, y - h/2) with THEME subgraph styles, rx=8
   - Label `<text>` at (x, y - h/2 + 20), THEME subgraph label styles
   - Return g

9. **createSVG(layout)** -- the main function:
   - Create `<svg>` element with `width` and `height` from layout (NO viewBox -- CSS transform handles zoom)
   - Create `<defs>`, call `createArrowMarkers(defs)`, append defs
   - Create a `<g class="smartb-diagram">` root group
   - Render subgraphs first (behind everything): `layout.subgraphs.forEach(renderSubgraph)`
   - Render edges next: `layout.edges.forEach(renderEdge)`
   - Render nodes last (on top): `layout.nodes.forEach(renderNode)`
   - Append all to root group, append to svg
   - Return svg element

10. **Public API**: `window.SmartBSvgRenderer = { createSVG: createSVG }`

Use `var` throughout, `'use strict'` in IIFE. All SVG creation via `createElementNS(NS, ...)`. File should be ~200-280 lines.
