    **2. Create `src/server/static.ts`** -- Static file serving utility:
    - Export a `MIME_TYPES` record mapping extensions to content types: `.html` -> `text/html; charset=utf-8`, `.js` -> `application/javascript; charset=utf-8`, `.css` -> `text/css; charset=utf-8`, `.json` -> `application/json; charset=utf-8`, `.svg` -> `image/svg+xml`, `.png` -> `image/png`, `.mmd` -> `text/plain; charset=utf-8`
    - Export `serveStaticFile(res: ServerResponse, filePath: string): Promise<boolean>` -- reads file with `fs/promises`, sets Content-Type from MIME_TYPES (fallback `application/octet-stream`), writes 200 response, returns true. Returns false on error (file not found).
    - Use `node:fs/promises` readFile and `node:path` extname. Follow ESM import pattern with `.js` extensions.

    **3. Create `src/server/server.ts`** -- Main server module:
    - Import from `node:http` (createServer, IncomingMessage, ServerResponse), `detect-port` (detect), `open` (default import)
    - Export `ServerOptions` interface: `{ port: number; dir: string; openBrowser: boolean }`
    - Export `startServer(options: ServerOptions): Promise<void>`:
      a. Create `DiagramService` instance with `path.resolve(options.dir)` as project root
      b. Call `detect(options.port)` to find available port. If different from requested, log warning via `log.warn`
      c. Create thin router function:
         - Parse URL with `new URL(req.url ?? '/', \`http://\${req.headers.host}\`)`
         - Set CORS headers on ALL responses via helper: `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS`, `Access-Control-Allow-Headers: Content-Type`
         - Handle OPTIONS preflight: return 204 with CORS headers immediately
         - Match route from registered routes (method + pathname pattern). Use `registerRoutes()` from routes.ts to get route array
         - If no route matches, try serving static file from `getStaticDir()` (map URL pathname to file in static dir). Use path traversal protection.
         - If URL pathname is `/` or `/index.html`, serve `live.html` from static dir
         - If nothing matches, return 404 JSON `{ error: 'Not Found' }`
      d. Create HTTP server with `createServer(handler)`
      e. `server.listen(actualPort, () => { ... })` -- log URL, open browser if `options.openBrowser` is true (catch and warn on open failure)
      f. Register SIGINT handler for graceful shutdown: `server.close(() => process.exit(0))`

    - Export helper `sendJson(res: ServerResponse, data: unknown, status?: number): void` -- stringify data, set Content-Type and Content-Length, write response with CORS headers
    - Export helper `readJsonBody<T>(req: IncomingMessage): Promise<T>` -- collect chunks from request stream, parse as JSON
    - Export route type: `interface Route { method: string; pattern: RegExp; handler: Handler }` and `type Handler = (req: IncomingMessage, res: ServerResponse, params: Record<string, string>) => void | Promise<void>`

    **Important patterns to follow:**
    - Use `log` from `../utils/logger.js` (stderr-only -- never console.log)
    - Use `getStaticDir()` from `../utils/paths.js` for static asset root
    - Two separate file roots: static assets from `getStaticDir()`, diagrams from user's project dir (DiagramService handles this)
    - ESM imports with `.js` extension suffixes

    **4. Add `serve` subcommand to `src/cli.ts`:**
    - Add commander `.command('serve')` with `.description('Start the diagram viewer server')`
    - Options: `-p, --port <number>` (default `'3333'`), `-d, --dir <path>` (default `'.'`), `--no-open` (disable browser auto-open)
    - Action handler: use dynamic `import()` to lazy-load `./server/server.js`, call `startServer({ port: parseInt(options.port, 10), dir: options.dir, openBrowser: options.open })`
    - Dynamic import keeps server code out of initial CLI load time

    **Routes to register:**

    1. **GET /tree.json** -- File tree for sidebar
       - Call `service.listFiles()` to get flat list of relative .mmd paths
       - Transform to nested tree structure using `buildFileTree()` helper (defined in this file)
       - Return JSON array via `sendJson`
       - `buildFileTree(files: string[]): TreeNode[]` -- takes flat paths like `['folder/file.mmd', 'root.mmd']`, returns nested `[{ type: 'folder', name, children: [...] }, { type: 'file', name, path }]`

    2. **GET /*.mmd** (pattern: `/^\/(.+\.mmd)$/`) -- Serve raw .mmd file content
       - Extract file path from URL pathname (strip leading `/`)
       - Use `resolveProjectPath(projectDir, filePath)` for path traversal protection
       - Read file with `readFile(resolved, 'utf-8')` and return as `text/plain; charset=utf-8`
       - Return 404 if file not found

    3. **POST /save** -- Save diagram content
       - Parse body with `readJsonBody<{ filename: string; content: string }>(req)`
       - Use `resolveProjectPath(projectDir, body.filename)` for path security
       - Create parent dirs with `mkdir(dirname(resolved), { recursive: true })`
       - Write with `writeFile(resolved, body.content, 'utf-8')`
       - Return `sendJson(res, { ok: true })`
       - Return 400 on missing filename/content, 500 on write error

    4. **POST /delete** -- Delete diagram file
       - Parse body with `readJsonBody<{ filename: string }>(req)`
       - Resolve with path traversal protection
       - Delete with `unlink(resolved)`
       - Return `sendJson(res, { ok: true })`
       - Return 404 if file not found

    5. **POST /mkdir** -- Create directory
       - Parse body with `readJsonBody<{ folder: string }>(req)`
       - Resolve with path traversal protection
       - Create with `mkdir(resolved, { recursive: true })`
       - Return `sendJson(res, { ok: true })`

    6. **POST /move** -- Rename/move file
       - Parse body with `readJsonBody<{ from: string; to: string }>(req)`
       - Resolve both paths with traversal protection
       - Create target parent dirs with `mkdir(dirname(resolvedTo), { recursive: true })`
       - Move with `rename(resolvedFrom, resolvedTo)`
       - Return `sendJson(res, { ok: true })`

    7. **GET /api/diagrams** -- REST: List all diagrams (HTTP-02)
       - Call `service.listFiles()`
       - Return `sendJson(res, { files: fileList })`

    8. **GET /api/diagrams/:file** -- REST: Get diagram content (HTTP-03)
       - Extract `:file` param from URL (matched via regex named group `(?<file>.+)`)
       - URL-decode the file param
       - Call `service.readDiagram(file)` to get full DiagramContent
       - Return JSON with: `{ filePath, mermaidContent, flags: Object.fromEntries(diagram.flags), validation: { valid, errors, diagramType } }`
       - Note: Convert Map to plain object for JSON serialization
       - Return 404 if file not found (catch readDiagram error)

    **Helper function in this file:**

    ```typescript
    interface TreeNode {
      type: 'file' | 'folder';
      name: string;
      path?: string;
      children?: TreeNode[];
    }

    function buildFileTree(files: string[]): TreeNode[] {
      const root: TreeNode[] = [];
      for (const filePath of files) {
        const parts = filePath.split('/');
        let current = root;
        for (let i = 0; i < parts.length; i++) {
          const part = parts[i]!;
          const isFile = i === parts.length - 1;
          if (isFile) {
            current.push({ type: 'file', name: part, path: filePath });
          } else {
            let folder = current.find(n => n.type === 'folder' && n.name === part);
            if (!folder) {
              folder = { type: 'folder', name: part, children: [] };
              current.push(folder);
            }
            current = folder.children!;
          }
        }
      }
      return root;
    }
    ```

    **Error handling pattern for all POST routes:**
    ```typescript
    try {
      // ... operation
      sendJson(res, { ok: true });
    } catch (err) {
      const message = err instanceof Error ? err.message : 'Unknown error';
      sendJson(res, { error: message }, err?.code === 'ENOENT' ? 404 : 500);
    }
    ```

    **Pattern for route registration:**
    Each route uses the `Route` interface from server.ts. The `pattern` field is a RegExp. For simple paths like `/save`, the pattern is `new RegExp('^/save$')`. For parameterized paths like `/api/diagrams/:file`, use `new RegExp('^/api/diagrams/(?<file>.+)$')`. For .mmd file paths, use `new RegExp('^/(.+\\.mmd)$')`.

    The params from named regex groups are passed to the handler via the `params` argument.
