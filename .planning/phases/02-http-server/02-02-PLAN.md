    **1. Extend `static/annotations.js` with status annotation support:**

    The annotation system already supports `%% @flag nodeId "message"`. Add parallel support for `%% @status nodeId statusValue` where statusValue is one of: `ok`, `problem`, `in-progress`, `discarded`.

    - Add `STATUS_REGEX` constant: `/^%%\s*@status\s+(\S+)\s+(\S+)$/`
    - In `parseAnnotations(content)`, after checking for flag matches, also check for status matches. Store in a separate `statuses` Map (nodeId to statusValue string)
    - Add to `state` object: `statuses: new Map()` alongside existing `flags`
    - Export `getStatusMap()` function via `SmartBAnnotations` that returns the current statuses map
    - In `serializeAnnotations()`, also serialize status entries as `%% @status nodeId statusValue` after flag entries
    - Add `setStatus(nodeId, status)` and `removeStatus(nodeId)` functions exposed on `SmartBAnnotations`
    - In the `sync()` function where flags are parsed from content, also parse and update statuses

    **2. Modify `static/live.html` render function for status color injection (UI-04):**

    In the `render(code)` function, AFTER stripping annotations but BEFORE calling `mermaid.render()`, inject classDef definitions and class assignments.

    Add an `injectStatusStyles(cleanCode)` function in the script section:
    - If `window.SmartBAnnotations` is not available or `getStatusMap()` returns empty map, return cleanCode unchanged
    - Append classDef definitions to the clean Mermaid content:
      - `classDef ok fill:#22c55e,stroke:#16a34a,color:#fff;`
      - `classDef problem fill:#ef4444,stroke:#dc2626,color:#fff;`
      - `classDef inProgress fill:#eab308,stroke:#ca8a04,color:#000;`
      - `classDef discarded fill:#9ca3af,stroke:#6b7280,color:#fff;`
    - For each entry in the status map, append `class nodeId className` where className maps: `ok` to `ok`, `problem` to `problem`, `in-progress` to `inProgress`, `discarded` to `discarded`
    - Only inject if there are actual status entries

    In the `render()` function, change the flow so that after getting `cleanCode`, call `injectStatusStyles(cleanCode)` to get `styledCode`, then render `styledCode` instead of `cleanCode`.

    **3. Enhance error display with line numbers (UI-07):**

    Replace the existing error catch block in `render()` with a structured error panel. The current error display is a simple `<pre>` with the error message. Enhance it to:

    - Add an `escapeHtml(text)` helper function using `document.createElement('div')` + `textContent` + returning `div.innerHTML` for safe HTML escaping
    - Parse line numbers from Mermaid error messages using regex `/line\s+(\d+)/i`
    - Display error with: error icon (inline SVG), title "Mermaid Syntax Error", line number badge (if available), escaped error message in a styled pre block
    - If line number is available, show a code snippet with surrounding context (3 lines before, 2 lines after the error), with the error line highlighted in red background
    - Use the `cleanCode` variable (before status injection) to show the code snippet, so the displayed source matches what the developer wrote
    - Style the error panel with: red accent color (#ef4444), white background, rounded corners, Inter font family, consistent with the existing UI style

    **Important notes:**
    - Use `document.createElement` + `textContent` for escaping user content to prevent XSS
    - The error display uses DOM manipulation with safe text content assignment where possible
    - Keep the existing UI language style (the current codebase has Portuguese strings in some places)
