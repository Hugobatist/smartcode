**src/diagram/types.ts:** Create all TypeScript types for the diagram domain. Follow the research examples exactly:
- `NodeStatus` type: `'ok' | 'problem' | 'in-progress' | 'discarded'`
- `Flag` interface: `{ nodeId: string; message: string; timestamp?: number }`
- `DiagramNode` interface: `{ id: string; label: string; shape: string; status?: NodeStatus }`
- `DiagramEdge` interface: `{ from: string; to: string; label?: string; type: 'arrow' | 'open' | 'dotted' | 'thick' }`
- `DiagramContent` interface: `{ raw: string; mermaidContent: string; flags: Map<string, Flag>; validation: ValidationResult; filePath: string }`
- `ValidationResult` interface: `{ valid: boolean; errors: ValidationError[]; diagramType?: string }`
- `ValidationError` interface: `{ message: string; line?: number; column?: number }`
- `Project` interface: `{ rootDir: string; files: string[] }`

Export ALL types. These are the foundation that every future module imports from.

**src/utils/logger.ts:** Create stderr-only logger using picocolors:
- `log.info(...)` -> `console.error(pc.blue('[smartb]'), ...args)`
- `log.warn(...)` -> `console.error(pc.yellow('[smartb]'), ...args)`
- `log.error(...)` -> `console.error(pc.red('[smartb]'), ...args)`
- `log.debug(...)` -> only if `process.env.DEBUG`, `console.error(pc.dim('[smartb]'), ...args)`
- NEVER use `console.log` -- stdout is reserved for MCP stdio transport in Phase 5.

**src/utils/paths.ts:** Create cross-platform path utilities:
- `getStaticDir()`: returns `path.join(import.meta.dirname, '..', 'static')` -- resolves to `dist/static/` when running from built output
- `getStaticFile(filename: string)`: returns `path.join(getStaticDir(), filename)`
- `resolveProjectPath(projectRoot: string, filePath: string)`: resolves a relative path against a project root and validates it does not escape via path traversal. Throws if the resolved path does not start with the project root.

**src/cli.ts:** Create CLI entry point:
- First line: `#!/usr/bin/env node`

- Import `log` from `./utils/logger.js` (note .js extension for ESM)
- Create program with name `'smartb'`, description `'AI observability diagrams -- see what your AI is thinking'`
- Read version from a hardcoded string `'0.1.0'` (not from package.json to avoid import assertion complexity)
- Add `.version('0.1.0')` to the program
- Call `program.parse()`
- No subcommands yet beyond the built-in `--version` and `--help`

**src/index.ts:** Create barrel export file:

- This file is the public API entry point. Downstream packages import from `smartb-diagrams`.
