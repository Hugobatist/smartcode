  <action>
Create `static/context-menu.js` as an IIFE module that shows a positioned HTML context menu on right-click over diagram nodes and edges.

**Dependencies:** `interaction-state.js` (SmartBInteraction), `diagram-dom.js` (DiagramDOM), `diagram-editor.js` (MmdEditor), `annotations.js` (SmartBAnnotations), `selection.js` (SmartBSelection).

**Core functions:**

1. `showContextMenu(x, y, nodeInfo)` — Builds and shows the context menu HTML overlay.
   - Clear any existing context menu (`closeContextMenu()`)
   - Create a `<div class="context-menu">` positioned at `(x, y)` using screen coordinates
   - Clamp position to stay on-screen: `left = Math.min(x, window.innerWidth - 200)`, `top = Math.min(y, window.innerHeight - 250)`
   - Build menu items based on `nodeInfo.type`:

   **For nodes (`nodeInfo.type === 'node'` or `'subgraph'`):**
   | Label | Icon hint | Action |
   |-------|-----------|--------|
   | Editar Texto | pencil | Opens inline edit via `SmartBInlineEdit.open(nodeInfo.id)` |
   | Deletar | trash | Calls `MmdEditor.doRemoveNode(nodeInfo.id)` |
   | Duplicar | copy | Calls `doDuplicate(nodeInfo.id)` — see below |
   | Sinalizar | flag | Enters flag mode, shows flag popover for this node |
   | Nova Seta | arrow | Calls `MmdEditor.startConnectFrom(nodeInfo.id)` |

   **For edges (`nodeInfo.type === 'edge'`):**
   | Label | Action |
   |-------|--------|
   | Deletar | Calls edge deletion (parse edge endpoints from ID, call `MmdEditor.doRemoveEdge()`) |
   | Sinalizar | Enters flag mode, shows flag popover for this edge |

   - Each menu item is a `<div class="context-menu-item">` with a click handler
   - Menu items constructed with `document.createElement` (NOT innerHTML — project convention)
   - Add separator `<div class="context-menu-separator">` between Duplicar and Sinalizar

2. `closeContextMenu()` — Removes the context menu from DOM if it exists. Transitions FSM via `SmartBInteraction.transition('close')`.

3. `doDuplicate(nodeId)` — A helper that:
   - Gets editor content via `document.getElementById('editor').value`
   - Calls `MmdEditor.duplicateNode(content, nodeId)` (new function from Task 3) to get updated content
   - Applies the edit through a simple applyEdit-style pattern (strip annotations, apply, re-inject, save, re-render)
   - OR simpler: call MmdEditor's internal `applyEdit` if it's exposed. If not, create a minimal wrapper.
   - IMPORTANT: After duplicate, select the new node.

**Right-click handler (delegated on `#preview-container`):**
- Listen for `contextmenu` event
- If `SmartBInteraction.isBlocking()`, return
- If FSM state is `flagging`, `add-node`, or `add-edge`, return
- Use `DiagramDOM.extractNodeId(e.target)` to detect what was right-clicked
- If nodeInfo found: `e.preventDefault()`, select the node first (`SmartBSelection.selectNode()`), transition FSM to `context-menu`, show menu
- If no nodeInfo: let browser default context menu show

**Outside click handler:** Same pattern as flag popover — setTimeout 50ms, then add `mousedown` listener that closes menu if click is outside.

**Escape handler:** Listen on document `keydown` — if Escape and FSM state is `context-menu`, close menu.

**CSS (self-injected via style element at IIFE load, same pattern as selection.js):**
```css
.context-menu {
  position: fixed; z-index: 10000; background: var(--panel-bg, #1e1e2e);
  border: 1px solid var(--border, #313244); border-radius: 8px;
  box-shadow: 0 8px 24px rgba(0,0,0,0.4); padding: 4px 0;
  min-width: 180px; font-family: 'Inter', sans-serif; font-size: 13px;
}
.context-menu-item {
  padding: 8px 16px; cursor: pointer; color: var(--text, #cdd6f4);
  display: flex; align-items: center; gap: 8px; transition: background 0.1s;
}
.context-menu-item:hover { background: var(--hover-bg, #313244); }
.context-menu-item.danger { color: #ef4444; }
.context-menu-item.danger:hover { background: rgba(239,68,68,0.15); }
.context-menu-separator { height: 1px; background: var(--border, #313244); margin: 4px 0; }
```

**Public API (`window.SmartBContextMenu`):**
- `init()` — attaches contextmenu handler to `#preview-container`
- `close()` — closes the context menu
- `show(x, y, nodeInfo)` — shows context menu at position

  <name>Task 2: Create inline-edit.js — Double-click contenteditable overlay for label editing</name>
  <files>static/inline-edit.js</files>
  <action>
Create `static/inline-edit.js` as an IIFE module that opens an inline contenteditable HTML overlay when a node label is double-clicked, allowing in-place text editing.

**Dependencies:** `interaction-state.js` (SmartBInteraction), `diagram-dom.js` (DiagramDOM), `diagram-editor.js` (MmdEditor), `event-bus.js` (SmartBEventBus).

**Core functions:**

1. `open(nodeId)` — Opens the inline edit overlay for a node.
   - Find the node's `<text>` element via `DiagramDOM.findNodeElement(nodeId)` then `.querySelector('text')`
   - If no text element found, abort
   - Get screen-space position: `textEl.getBoundingClientRect()`
   - Get container position: `document.getElementById('preview-container').getBoundingClientRect()`
   - Get current label text: `DiagramDOM.getNodeLabel(nodeId)` or from `MmdEditor.getNodeText(editor.value, nodeId)`
   - Create overlay `<div>`:
     ```javascript
     var overlay = document.createElement('div');
     overlay.className = 'inline-edit-overlay';
     overlay.contentEditable = 'true';
     overlay.spellcheck = false;
     overlay.textContent = currentLabel;
     ```
   - Position relative to preview-container:
     ```javascript
     overlay.style.position = 'absolute';
     overlay.style.left = (textRect.left - containerRect.left) + 'px';
     overlay.style.top = (textRect.top - containerRect.top) + 'px';
     overlay.style.minWidth = Math.max(textRect.width, 60) + 'px';
     overlay.style.minHeight = textRect.height + 'px';
     ```
   - Style to match SVG text appearance:
     ```javascript
     var computed = window.getComputedStyle(textEl);
     overlay.style.fontFamily = computed.fontFamily || "'Inter', sans-serif";
     overlay.style.fontSize = computed.fontSize || '14px';
     overlay.style.fontWeight = computed.fontWeight || '400';
     overlay.style.color = '#cdd6f4'; // theme text color
     overlay.style.background = 'var(--panel-bg, #1e1e2e)';
     overlay.style.border = '2px solid #6366f1';
     overlay.style.borderRadius = '4px';
     overlay.style.padding = '2px 6px';
     overlay.style.outline = 'none';
     overlay.style.zIndex = '10001';
     overlay.style.whiteSpace = 'nowrap';
     ```
   - Append to `#preview-container`
   - Focus and select all text:
     ```javascript
     overlay.focus();
     var range = document.createRange();
     range.selectNodeContents(overlay);
     var sel = window.getSelection();
     sel.removeAllRanges();
     sel.addRange(range);
     ```
   - Store reference: `activeOverlay = overlay; activeNodeId = nodeId; originalLabel = currentLabel;`
   - Hide the original SVG text element: `textEl.style.visibility = 'hidden';`
   - Emit `edit:started` on EventBus: `{ nodeId: nodeId }`

2. `confirm()` — Saves the edit and closes.
   - Get new text: `activeOverlay.textContent.trim()`
   - If new text differs from `originalLabel` and is not empty:
     - Call `MmdEditor.doEditNodeText(nodeId)` — BUT wait, `doEditNodeText` uses `prompt()` which we want to BYPASS
     - INSTEAD: Use `MmdEditor.applyEdit()` directly if possible, or replicate the pattern:
       ```javascript
       var editor = document.getElementById('editor');
       var content = editor.value;
       // Strip annotations, apply edit, re-inject
       var annotations = window.SmartBAnnotations;
       var flags = new Map();
       if (annotations) { flags = annotations.getState().flags; content = annotations.stripAnnotations(content); }
       content = MmdEditor.editNodeText(content, activeNodeId, newText);
       if (annotations) content = annotations.injectAnnotations(content, flags);
       editor.value = content;
       // Save and re-render (use the same hook pattern)
       ```
     - SIMPLER APPROACH: Call `MmdEditor.editNodeText(content, nodeId, newText)` directly as a pure function (it just does string manipulation), then save/re-render through the hooks.
   - Close the overlay
   - Transition FSM: `SmartBInteraction.transition('confirm')`
   - Emit `edit:completed`: `{ nodeId, oldLabel: originalLabel, newLabel: newText }`

3. `cancel()` — Discards changes and closes.
   - Restore SVG text visibility if possible (but SVG will be re-rendered anyway after save)
   - Close overlay
   - Transition FSM: `SmartBInteraction.transition('cancel')`
   - Emit `edit:cancelled`: `{ nodeId: activeNodeId }`

4. `close()` — Removes overlay from DOM, resets internal state.

**Event handlers on the overlay:**
- `keydown`:
  - Enter (without Shift): `e.preventDefault(); confirm();`
  - Escape: `cancel();`
- `blur`: Treat as confirm (same as clicking away)
  - IMPORTANT: Use a small setTimeout(0) to avoid blur firing when user clicks a context menu item
  - `setTimeout(function() { if (activeOverlay) confirm(); }, 0);`

**Double-click handler (delegated on `#preview-container`):**
- Listen for `dblclick` event
- If `SmartBInteraction.isBlocking()`, return
- If FSM state is `flagging`, `add-node`, `add-edge`, return
- Use `DiagramDOM.extractNodeId(e.target)` to detect what was double-clicked
- If nodeInfo and type is `node` or `subgraph`: transition FSM `dbl_click`, call `open(nodeInfo.id)`
- If edge: ignore (edges can't be inline-edited)

**EventBus subscription:**
- Subscribe to `diagram:rendered`: If inline edit is active, commit the current edit before the SVG is replaced (call `confirm()` if overlay exists). This prevents losing edits when the file changes via WebSocket.

**Public API (`window.SmartBInlineEdit`):**
- `init()` — attaches dblclick handler to `#preview-container`, EventBus subscription
- `open(nodeId)` — opens inline edit for a node
- `close()` — closes without saving
- `confirm()` — saves and closes
- `cancel()` — discards and closes
- `isActive()` — returns true if an inline edit overlay is currently open

**Target: ~200-260 lines.**

**IMPORTANT interaction with collapse-ui.js:** collapse-ui.js currently uses `dblclick` on `#preview` for entering focus mode. The inline-edit dblclick handler must:
1. Check if the clicked node is a `__collapsed__` node — if so, DON'T open inline edit (let collapse handle it)
2. Call `e.stopPropagation()` after opening inline edit to prevent collapse-ui from also firing
3. The handler on `#preview-container` fires before `#preview` (due to event bubbling), so attach to `#preview-container` and stop propagation.
