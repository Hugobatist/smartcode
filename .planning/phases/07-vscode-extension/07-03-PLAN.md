    **status-bar.ts** — StatusBarManager:

    Implement exactly as described in 07-RESEARCH.md Pattern 3:
    - Export class `StatusBarManager`.
    - Constructor creates a `vscode.StatusBarItem` via `vscode.window.createStatusBarItem('smartb.connectionStatus', vscode.StatusBarAlignment.Right, 100)`.
    - Set `item.name = 'SmartB Connection'` and `item.command = 'smartb.reconnect'`.
    - Initial state: call `setStatus('disconnected')`.
    - Call `item.show()`.
    - `setStatus(status: 'connected' | 'disconnected' | 'reconnecting')` method:
      - connected: text `$(check) SmartB`, tooltip `SmartB: Connected to server`, backgroundColor undefined
      - disconnected: text `$(error) SmartB`, tooltip `SmartB: Disconnected (click to reconnect)`, backgroundColor `new vscode.ThemeColor('statusBarItem.errorBackground')`
      - reconnecting: text `$(sync~spin) SmartB`, tooltip `SmartB: Reconnecting...`, backgroundColor `new vscode.ThemeColor('statusBarItem.warningBackground')`
    - `dispose()` method: calls `item.dispose()`.
    - Keep under ~60 lines.

    **Update extension.ts**:

    1. Import StatusBarManager.
    2. In activate(): create `const statusBar = new StatusBarManager()`.
    3. Wire WsClient onStatus callback to also call `statusBar.setStatus(status)` in addition to relaying to webview.
    4. Push statusBar to context.subscriptions for disposal.
    5. Ensure extension.ts stays under 200 lines. If it grows beyond that, extract the WebSocket message handling and flag saving into a separate module (e.g., `message-handler.ts`).

    **README.md** — Marketplace description:

    Create a concise README for the VS Code Marketplace:
    - Title: SmartB Diagrams
    - Tagline: "Live AI reasoning diagrams in your VS Code sidebar"
    - Features section (bulleted):
      - Live Mermaid diagram rendering in sidebar panel
      - Real-time updates via WebSocket (no manual refresh)
      - Click nodes to add flag annotations for AI correction
      - Connection status indicator in status bar
      - Automatic reconnection with exponential backoff
    - Requirements section:
      - A running SmartB server (`npx smartb-diagrams serve`)
      - Node.js 22 or later
    - Getting Started section:
      1. Install the extension
      2. Start SmartB server: `npx smartb-diagrams serve`
      3. Open the SmartB sidebar (click the icon in the activity bar)
      4. The diagram appears automatically
    - Configuration section:
      - `smartb.serverUrl`: WebSocket URL (default: ws://localhost:3333/ws)
      - `smartb.autoConnect`: Auto-connect on startup (default: true)
    - Commands section:
      - SmartB: Reconnect to Server
      - SmartB: Open in Browser
