<objective>
Add file name header, file selector dropdown, and multi-file navigation to the VS Code webview panel so developers can see which diagram they're viewing and switch between multiple .mmd files.

Purpose: The current panel shows a diagram without any context — no file name, no way to switch files. This makes the extension unusable when a project has multiple .mmd diagrams. Adding navigation completes the core VS Code experience.

Output: A webview panel with a header showing the current file name, a dropdown to select from available .mmd files, and automatic updates when files are added/removed.
</objective>

         if (msg.type === 'selectFile') {
           const file = msg.file;
           // Check if we have cached content
           if (fileContents.has(file)) {
             provider.postMessage({
               type: 'diagram:update',
               data: { type: 'file:changed', file, content: fileContents.get(file) }
             });
           } else {
             // Fetch via HTTP
             const content = await httpGet(`${httpBaseUrl}/api/diagrams/${encodeURIComponent(file)}`);
             if (content) {
               fileContents.set(file, content);
               provider.postMessage({
                 type: 'diagram:update',
                 data: { type: 'file:changed', file, content }
               });
             }
           }
           currentFile = file;
         }
