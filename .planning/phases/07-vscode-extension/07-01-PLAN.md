    **ws-client.ts** — WebSocket client for extension host (Node.js context):
    - Export class `SmartBWsClient` with constructor(serverUrl: string, callbacks: { onMessage: (msg: any) => void, onStatus: (status: 'connected' | 'disconnected' | 'reconnecting') => void })
    - `connect()` method: creates WebSocket instance using `ws` library. On 'open' → call onStatus('connected'), reset attempt counter. On 'message' → JSON.parse data.toString(), call onMessage. On 'close' → call onStatus('disconnected'), schedule reconnect. On 'error' → no-op (close fires after error).
    - Reconnection with exponential backoff: BASE_DELAY 500ms, MAX_DELAY 16000ms, jitter 50-100% of delay (same algorithm as static/ws-client.js but using Node.js setTimeout).
    - `disconnect()` method: clear reconnect timer, close WebSocket, set a `disposed` flag to prevent reconnection.
    - `reconnect()` method: disconnect then connect.
    - `isConnected()` getter: returns true if ws.readyState === WebSocket.OPEN.
    - Guard against multiple connections (don't create new WS if one is already open/connecting).
    - Import WebSocket from 'ws' (Node.js library, not browser API).
    - Keep under ~120 lines. This mirrors the logic in static/ws-client.js but adapted for Node.js.

    **diagram-provider.ts** — WebviewViewProvider:
    - Export class `DiagramViewProvider` implementing `vscode.WebviewViewProvider`.
    - Static readonly `viewType = 'smartb.diagramView'` (must match package.json views id).
    - Constructor takes `extensionUri: vscode.Uri`.
    - `resolveWebviewView(webviewView, context, token)`:
      - Store reference to `this.view = webviewView`.
      - Set webview.options: enableScripts true, localResourceRoots [vscode.Uri.joinPath(extensionUri, 'media')].
      - Set webview.html using `getHtmlForWebview()` private method.
      - Listen for webview messages via `webview.onDidReceiveMessage(msg => ...)`. Forward messages to an optional `onWebviewMessage` callback.
      - On `webviewView.onDidDispose(() => this.view = undefined)`.
    - `getHtmlForWebview(webview)` private method:
      - Generate crypto-random nonce (32 chars from possible alphanumeric characters).
      - Build CSP: default-src 'none', style-src ${webview.cspSource} 'unsafe-inline', script-src 'nonce-${nonce}', img-src ${webview.cspSource} data:, font-src ${webview.cspSource}.
      - Get URIs for media/webview.css, media/webview.js, media/mermaid.min.js using webview.asWebviewUri().
      - Return HTML template with CSP meta tag, stylesheet link, body with #diagram div and #connection-status span, scripts with nonce.
      - NOTE: media/mermaid.min.js won't exist yet — that's plan 07-02. The HTML template should reference it but it won't be available until 07-02 downloads it.
    - `postMessage(message: unknown)` public method: if this.view exists, call this.view.webview.postMessage(message).
    - `isVisible()` getter: returns this.view !== undefined.
    - Keep under ~130 lines.

    **extension.ts** — Extension entry point:
    - Export `activate(context: vscode.ExtensionContext)`:
      1. Create DiagramViewProvider instance with context.extensionUri.
      2. Register it: context.subscriptions.push(vscode.window.registerWebviewViewProvider(DiagramViewProvider.viewType, provider)).
      3. Read smartb.serverUrl from workspace configuration (vscode.workspace.getConfiguration('smartb').get('serverUrl')).
      4. Read smartb.autoConnect from config.
      5. Create SmartBWsClient with serverUrl and callbacks:
         - onMessage: call provider.postMessage({ type: 'diagram:update', ...msg }) to relay WS messages to webview.
         - onStatus: call provider.postMessage({ type: 'connection:status', status }) to relay connection state to webview.
      6. If autoConnect is true, call wsClient.connect().
      7. Register smartb.reconnect command: wsClient.reconnect().
      8. Register smartb.openBrowser command: extract HTTP URL from WS URL (replace ws:// with http://, remove /ws path), open with vscode.env.openExternal.
      9. Listen for config changes: vscode.workspace.onDidChangeConfiguration(e => { if e.affectsConfiguration('smartb.serverUrl'), reconnect with new URL }).
      10. Push wsClient into disposables: context.subscriptions.push({ dispose: () => wsClient.disconnect() }).
    - Export `deactivate()`: empty function (cleanup handled by disposables).
    - Keep under ~100 lines.

    Also create a placeholder **vscode-extension/src/webview/main.ts** with a minimal IIFE that acquires the VS Code API and listens for postMessage events. It should:
    - Call `acquireVsCodeApi()`.
    - Restore state with `vscode.getState() || {}`.
    - Listen for 'message' events on window.
    - On 'diagram:update': set innerHTML of #diagram to a placeholder "Diagram received" message (actual mermaid rendering is plan 07-02).
    - On 'connection:status': update #connection-status text.
    - This file will be fully implemented in 07-02 — for now it's a working skeleton that proves the postMessage pipeline works.
