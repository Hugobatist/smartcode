  <action>
Create `static/search.js` — an IIFE module exposing `window.SmartBSearch` following the same pattern as `annotations.js`:

```
(function () {
    'use strict';
    // State
    const state = { isOpen: false, query: '', matches: [], currentIndex: -1 };
    let hooks = { ... };

    // Functions:
    // init(hooks) — store hooks reference (same pattern as SmartBAnnotations.init)
    // open() — show search bar, focus input, prevent default browser Ctrl+F
    // close() — hide search bar, clear highlights, reset state
    // search(query) — find matching nodes in current SVG
    // navigateNext() — move to next match
    // navigatePrev() — move to previous match
    // highlightMatches() — add CSS class to matching SVG nodes
    // scrollToMatch(index) — pan/zoom to center the matched node in view

    window.SmartBSearch = { init, open, close, getState: () => state };
})();
```

**Search logic:**
- Query `#preview svg` for all elements with class `nodeLabel` (Mermaid's label class for flowchart nodes)
- Also query `.cluster-label .nodeLabel` for subgraph labels
- For each, compare `element.textContent.toLowerCase()` against `query.toLowerCase()` (substring match)
- Collect matching node elements. Store the parent node group element (closest `.node` or `.cluster`) as the match target for highlighting and navigation
- A match count indicator like "2 of 5" should appear in the search bar

**Highlighting:**
- Add a CSS class `search-match` to all matching node groups (the `.node` or `.cluster` parent)
- Add a CSS class `search-match-active` to the currently focused match
- `search-match` gets a semi-transparent highlight outline
- `search-match-active` gets a stronger highlight (similar to the flag blink animation approach)

**Navigation:**
- Enter or down-arrow button advances `currentIndex`, wraps around
- Shift+Enter or up-arrow button goes back, wraps around
- On navigation, remove `search-match-active` from old match, add to new match
- Pan the view so the active match is centered in the viewport (use the match element's `getBoundingClientRect()` and adjust panX/panY/zoom via hooks)

**Pan-to-match logic:**
- Get the matched SVG element's bounding rect relative to the container
- Calculate the panX/panY needed to center that element in the preview-container
- Use the existing zoom value (don't change zoom)
- The hooks object should provide: `getPan: () => ({ panX, panY, zoom })`, `setPan: (px, py) => { panX = px; panY = py; applyTransform(); }`
- Access these via hooks set during init

**Search bar UI (built with DOM methods, not innerHTML — XSS safe):**
- A horizontal bar at the top of the preview panel (position absolute, z-index 50)
- Input field (autofocused when opened)
- Match count label ("N of M")
- Up/Down navigation buttons
- Close button (X)
- Input event on the text field triggers `search()` with debounce of 150ms
- Keydown on input: Enter = next, Shift+Enter = prev, Escape = close

Create `static/search.css`:
- `.search-bar` — positioned at top of preview panel, glassmorphism style matching zoom controls (var(--glass), backdrop-filter, rounded corners)
- `.search-bar input` — dark-on-glass input styling
- `.search-bar .match-count` — subtle count label
- `.search-bar button` — icon buttons for nav and close
- `.search-match rect, .search-match circle, .search-match polygon` — `stroke: #f59e0b !important; stroke-width: 3px !important;` (amber highlight)
- `.search-match-active rect, .search-match-active circle, .search-match-active polygon` — `stroke: #6366f1 !important; stroke-width: 4px !important; filter: drop-shadow(0 0 6px rgba(99,102,241,0.5));` (accent purple glow)
