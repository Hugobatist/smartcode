Fix THREE issues in static/live.html:

**1. PNG Export (Canvas Taint Fix)**

Replace the existing `exportPNG()` function (around line 1132-1147) with a version that:
- Temporarily calls `mermaid.initialize()` with `flowchart: { htmlLabels: false }` (all other config same as the existing mermaid.initialize call at top of script)
- Calls `mermaid.render('export-png-' + Date.now(), styledCode.trim())` to produce a Canvas-safe SVG without foreignObject elements
- Uses the resulting SVG string (not the DOM SVG) to create the Image for Canvas drawImage
- Wraps in try/catch: on taint error, show toast('Erro ao exportar PNG — tente SVG')
- After export completes (success or fail), re-initialize mermaid with the original config (`htmlLabels: true`) so subsequent renders are unaffected
- Get the styledCode the same way the existing `render()` function does: use SmartBAnnotations.getCleanContent(editor.value), then injectStatusStyles()
- Make `exportPNG()` async since mermaid.render is async

**2. File Tree XSS Fix**

In the `renderNodes()` function (around line 993-1026), sanitize all user-controlled strings before HTML interpolation:
- Wrap `prettyFolder(n.name)` calls with `escapeHtml()`: `escapeHtml(prettyFolder(n.name))`
- Wrap `prettyName(n.name)` calls with `escapeHtml()`: `escapeHtml(prettyName(n.name))`
- Wrap `path` in onclick attributes with `escapeHtml()`: escape single quotes in path strings used in `onclick="loadFile('...')"` — use `.replace(/'/g, "\\'")` on the path before interpolation, or better, use `escapeHtml(path)` in the title attribute AND sanitize the onclick path
- The `escapeHtml()` function already exists (around line 639). Use it.

**3. Zoom Preservation on Live Updates**

The current `render()` function always calls `requestAnimationFrame(() => zoomFit())` after successful render (line ~819). This means WebSocket live updates reset zoom.

Fix:
- Add a module-level variable `let isInitialRender = true;` alongside the existing state variables
- In the `render()` function, after successful mermaid.render and applyTransform and applyFlagsToSVG, check `isInitialRender`:
  - If `isInitialRender` is true: call `requestAnimationFrame(() => zoomFit())` and set `isInitialRender = false`
  - If `isInitialRender` is false: do NOT call zoomFit — instead just call `applyTransform()` (zoom/pan values are already set from before)
- In `loadFile()` function, set `isInitialRender = true` before calling `syncFile()` — so navigating to a different file triggers a zoomFit
- In the init IIFE where the first render happens, `isInitialRender` is already true so zoomFit will fire naturally
- Keep the `zoomFit()` calls in toggle editor/sidebar click handlers unchanged (those are explicit user-triggered layout changes)
